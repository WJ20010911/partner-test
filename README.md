# 伴侣匹配测试

一套基于情景题目的恋爱价值观自测系统。用户通过回答一系列情景题，系统根据选项分值加权计算，得出恋爱匹配度评分。

## 题库分类

题目按以下维度分类，答题时系统从各分类中随机抽取：

| 分类 | 说明 |
|------|------|
| **常识与思维弹性** | 考察生活常识、逻辑思维、反诈意识与思维变通能力 |
| **人性底线与共情基准** | 考察对暴力、霸凌、隐私、身体自主权等基本人道的立场 |
| **日常摩擦中的本能反应** | 模拟日常相处中的分歧场景，考察情绪管理与沟通模式 |
| **对"关系现实感"的认知** | 考察对长期关系中的平淡、衰老、财务、责任等现实维度的认知 |
| **其他** | 无法归类的题目 |

每份测试包含 **12 道题**，从以上分类中随机抽取。

## 评分细则

### 评分规则

每题有 **4 个选项**，每个选项对应一个分数（0\~3 分）：

| 选项 | 含义 | 分值 |
|------|------|------|
| 0 分选项 | 最正确、最理想的反应 | 0 分（加分最少） |
| 1 分选项 | 较不理想的反应 | 1 分 |
| 2 分选项 | 较理想的反应 | 2 分 |
| 3 分选项 | 最离谱、最不理想的反应 | 3 分（加分最多） |

**分数是反着给的**。分值越低的选项代表越正确的选择，旨在防止答题人猜出真实评分逻辑。

### 权重系统

每道题有一个权重值（weight，默认 1.0），影响该题在总分中的占比：

```
实际得分 = 选项分值 × 权重
```

权重越高，该题对最终分数的影响越大。

### 最终分数计算

```
real_score = Σ(每题得分 × 权重)        # 加权总分，越低越好
surface_score = max(0, 100 - real_score)  # 展示给用户的分数，越高越好
```

- **real_score**：真实加权总分，数值越低代表匹配度越高
- **surface_score**：页面展示分数 = 100 减去真实分，直观体现匹配度百分比

### 超时惩罚

题目设有答题时限（10\~60 秒）。超时未作答（超过时限 +2 秒）的，该题按 10 分 × 权重计入总分。

### 投诉惩罚

用户投诉某题并跳过，该题按 15 分 × 权重计入总分。

### 分数范围

| 分数范围 | 参考含义 |
|----------|----------|
| 0\~20 分 | 非常匹配，价值观高度契合 |
| 20\~40 分 | 比较匹配，大多数方面契合 |
| 40\~60 分 | 一般匹配，部分方面存在差异 |
| 60\~80 分 | 不太匹配，核心方面有分歧 |
| 80\~100 分 | 差异较大，需要认真沟通 |
| 100 分以上 | 快跑吧               |

## 答题流程

1. 在首页点击 **"开始测试"**
2. 系统随机抽取 12 道题
3. 每题凭第一感觉选择一个答案
4. 可点击 **"跳过"** 跳过当前题（系统会补充一道新题）
5. 可点击 **"投诉并跳过"** 将题目标记为不适当（会触发投诉惩罚）
6. 提交后生成测试结果和验真码
7. 结果页可复制验真码，在验真页验证结果真伪

## 上传题目

### 上传方式

两种方式上传题目：

1. **注册账号后上传** — 使用邮箱注册，可设置用户名用于致谢展示
2. **以测试人员身份上传** — 跳过注册，点击"以测试人员身份进入"后输入昵称即可上传
   （注意，第二种方式只有在题库数量小于 200，或某维度题量小于 40 可用）

上传的题目需管理员审核后才能进入公共题库。

### 题目格式

每道题包含：
- **题干**：描述一个场景或问题
- **4 个选项**：每个选项需附带 0\~3 分的分值
- **分类**：选择对应的题目类别
- **答题时限**：10\~60 秒
- **标签**：逗号分隔的关键词，便于检索和归类

### 出题规范

- 情景应贴近真实的恋爱场景
- 选项应有明确的得分梯度（0 分为最正确，3 分为最不理想）
- 避免出现明显冒犯或歧视性内容
- 避免涉及政治、宗教等敏感话题

## 投诉通道

答题过程中如果遇到不适当的题目（如内容冒犯、表述不清、与恋爱无关等），可以点击 **"投诉并跳过"**。

投诉数据会聚合到管理后台，管理员可按投诉率查看：

| 投诉率 | 处理建议 |
|--------|----------|
| ≥ 10% 且投诉数 ≥ 5 | 高投诉率，建议优先处理，题目可能有问题 |
| 5% ~ 10% | 中等投诉率，建议关注 |
| < 5% | 低投诉率，可酌情处理 |

管理员可以直接在后台删除高投诉率题目。

## 验真机制

测试结果附带一个 HMAC-SHA256 签名的验真码，用于验证结果是否被篡改：

- 验真码包含：记录 ID、真实分数、时间戳、签名
- 用户可复制验真码，在 **验真页** 输入验证
- 验证通过则显示"验真通过 — 结果真实"
- 验证失败则提示"验真失败 — 结果可能被篡改"

验真码基于服务端 `SECRET_KEY` 签名，前端无法伪造。

访问 `/admin.html` 进入管理后台，需要输入管理员密码。

后台功能：

| 功能 | 说明 |
|------|------|

## 注意事项与免责条款

### 注意事项

1. **密码安全**：本站不要求使用高强度密码，但**强烈建议不要使用你在其他网站用过的密码**，防止被撞库攻击。密码仅使用 PBKDF2-SHA256 加盐哈希存储。
2. **注册无需验证邮箱**：注册时填写的邮箱仅用于登录识别，不会发送验证邮件。
3. **测试结果仅供参考**：本系统基于有限的题目和算法生成结果，不代表专业的性格或价值观评估。
4. **数据匿名性**：测试数据不关联用户身份，结果仅通过验真码验证，管理员无法查看某个具体答题人的详细信息。
5. **题目审核**：用户上传的题目需经管理员审核，但不保证 100% 的审核准确率。
6. **网络环境**：上传和测试功能需要网络连接，建议在网络稳定的环境下使用。

### 免责条款

1. **结果非专业评估**：本系统提供的测试结果仅供个人参考和娱乐，不构成任何形式的专业心理学、婚恋咨询或性格评估建议。
2. **不对等承诺**：测试结果不代表实际恋爱关系中的表现，也不对未来的伴侣选择或关系发展做任何预测或承诺。
3. **内容责任**：用户上传的题目内容由上传者自行负责，系统运营方不对其内容的准确性、合法性或适当性承担责任。
4. **投诉处理**：运营方会尽力处理用户投诉，但不保证能及时处理所有不当内容。
5. **服务可用性**：运营方尽力保持服务稳定，但不承担因维护、故障或不可抗力导致的访问中断责任。
6. **数据安全**：尽管采取了密码哈希等安全措施，运营方不保证数据绝对不受攻击。
7. **使用即同意**：使用本系统即视为同意以上条款。

## 技术栈

| 组件 | 技术 |
|------|------|
| 后端 | Python 3 标准库（http.server） |
| 数据库 | SQLite3（WAL 模式） |
| 前端 | 原生 HTML / CSS / JavaScript |
| 密码哈希 | PBKDF2-SHA256 + 随机 16 字节 salt |

| 用户认证 | JWT（HS256） |
| 外部依赖 | **零依赖**（纯 Python 标准库） |

## 部署

### Railway 部署

1. 推送代码到 GitHub 仓库
2. 在 Railway 新建项目，选择 GitHub 仓库
3. 设置环境变量（可选）:
   - `ADMIN_PASSWORD`：管理员密码
   - `SECRET_KEY`：验真签名密钥
